## Expected User Actions

**Vibe Coding (Hackathon) approach:**

**Hour 0-4: Ideation**
- Brainstorm 5-10 education + AI ideas
- Pick one with high demo appeal (visual, impactful)
- Sketch UI mockup (designer leads)

**Hour 4-12: Rapid prototyping**
- Use template (Bootstrap/Tailwind for speed)
- Integrate AI API (OpenAI, no custom models)
- Get something on screen (any output is progress)

**Hour 12-36: Feature building**
- Add 3-5 "wow" features (focus on judges' reaction)
- Fake anything that's slow (mock AI responses if API is slow)
- Ignore error handling (demo won't have errors)

**Hour 36-44: Polish for demo**
- Designer: Make it look professional
- Practice demo flow (rehearse 2-minute pitch)
- Fix only demo-breaking bugs

**Hour 44-48: Prep + rest**
- Write elevator pitch (30 seconds)
- Sleep (look alert for judging)

**What to skip entirely:**
- Tests (no time)
- Documentation (won't exist after weekend)
- Code review (trust each other)
- Refactoring (code quality irrelevant)
- Authentication (it's a demo)

## Hackathon-Specific Tips

**Demo appeal optimization:**
- Visual > functional (pretty UI beats boring command line)
- Story > technology (compelling narrative beats technical depth)
- Working flow > feature completeness (one complete flow > five half-done features)

**Technology choices:**
- Use what you know (learning new tech wastes time)
- Pre-made components (don't build from scratch)
- No-code tools are acceptable (Airtable, Zapier, etc.)

**Time management:**
- Cut scope ruthlessly (one good feature > ten mediocre ones)
- Avoid rabbit holes (if stuck for 30 min, move on)
- Sleep strategy (power nap at hour 32, alert for judging)
